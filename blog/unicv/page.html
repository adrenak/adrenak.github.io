Image Processing is one of those things that I like to think about. But, for some reason or the other setting up an OpenCV environment has always been a pain. I would say the same about OpenGL as well. Among a bunch of things that I am looking forward to this year is to work on things a bit outside Unity.

Not that I think Unity is going away anytime soon, it is probably only going to get stronger as we enter the VR/AR era and it almost becomes the defacto tool for those new experiences.

But, I want to explore more theoretical concepts. There is a lot that OpenCV has to offer. Probably it will be a good idea to start off OpenCV in Unity itself!

There is a C# library wrapper for OpenCV called OpenCVSharp which has been integrated into Unity. There are even some paid solutions for this but I haven't looked into them. The open source ones on the other hand seem a bit slow.

I tried one such project and got a pretty low 9FPS on average. With some optimizations I managed to get it to 20+ FPS which is much better.

Find the project here. It includes a sample Canny filter implementation. If things go as planned, I will add more features to this and eventually make OpenCV into an API-ish solution for image processing over videos, web cam feeds and more. There are some things to be solved, but I'll figure them along the way.

I've got quite interested in making some open tools recently. Much of them are actually just modules I require from time to time or can help me improve my workflow, but it's a great idea to have them available to all. Code sharing and collaboration is something I want to do more of, as I have mostly worked alone for the last 7 years or so.

Of late, I have changed my development methodology and no longer try to develop and design everything from scratch. The Unity Asset Store and Github are now an essential part of my daily routine. Thus, both the quality of my results and pace of learning by seeing other peoples' work has gone up.

Currently, I have 3 repositories on Github that I have released over the last 2 weeks. They are mini projects, tools that help with productivity and code reuse. I have a list of about 10 more projects I want to release, probably over the next 3 months and more are being added every now and then.

UniMic

A simple abstraction over the Microphone class for Unity. Good for microphone input, or recording voice. I used this for a fund raiser sort of event where firecrackers were activated on screen using voice. I have a couple of more interesting applications where I can use this.

BeneDict

If I had to wish for something for my birthday this year, it would probably be support for seriaziation for Dictionary in the Unity editor. Until then the Unify Community's SerializableDictionary project should suffice. However, declaring dictionary classes so they are serialized by it is still a pain. BeneDict automates the code generation.

Uni360Video

Official Unity blog has a guide on how to show 360 VR videos using RenderTexture and what seems like a Skybox shader. I'm not sure, but that sounds expensive for mobile devices. Another way is to have a headtracked camera in the center of a sphere with normals facing inwards. Such a setup requires manual work, Uni360Video automates this.

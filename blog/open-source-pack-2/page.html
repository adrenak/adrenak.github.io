Some more of my new open source code is out, although not all of them are complete, they are in a pretty functioning state. Almost all of them are things I need for my client or independent work.

UniPrep

The most important library I have made till date. This is the first one I import as soon as I start a new project. It offers a growing number of C# extensions, UnityEditor key bindings and a lot of utility scripts.

This thing is awesome and always under development. Check it out now.

UniGOAP

A library for Goal Oriented Action Plainning AI I have been working. I started making FSM based AI from the book Programming Game AI by Example but that soon started to fall short on the amount of features I wanted my agents to have. I need to simulate tribe members for a Live Simulation I am making for a client.

The GOAP approach was popularised by its award winning implementation in F.E.A.R. a game which I played for almost 2 years and was obsessed with it. I actually got pretty good at it. I'm playing Fortnite now with my flatmate and am really struggling ....

There is some thoery linked in UniGOAPS README and I highly recommend reading it.

Generic VR stuff

There is quite a lot of things out there for the Oculus Rift, HTV Vive and even GearVR. But one of my clients works with low cost VR and needs his applications for for Google Cardboard devices.

I looked around and could not find much that promises as much as NewtonVR and VRTK offer. Ofcourse Generic VR is quite primitive with no motion controllers or 6DOF but I require atleast some simple tools to quickly make gaze and joystick input based interactions.

The VRSamples project from Unity is brilliant but it does not work with Gaze based VR. GoogleVR SDK is quite feature rich but it has a lot of Daydream related code and I don't like the API or the way the prefabs have been structured. It does have an amazing InstantPreview tool though.

So I Made UniGenVR (Unity Generic VR). It has the best of both VRSamples and GoogleVR. VRSamples is modified to work with both gaze based and keyboard inputs. I also added new features to VRSamples to make it easier to work with. The InstantPreview from GoogleVR finally makes it easy to run things out of the editor itself. It's like the Unity Remote of VR.

I also separated out InstantPreview from GoogleVR although I'm not sure how useful that would be.

With UniGenVR I can finally not procastinate the development of generic VR applications like before just because of the sheer lack of good tools out there.

PS: VRSamples has some AMAZING use of coroutines. If only Unity's Coroutine API was any good. Well there is a Coroutine manager in UniPrep and one of the things I am writing soon is how to unleash the power of Coroutines in Unity.

UniCV

A small side project for doing OpenCV manipulations on Unity textures. One of the reasons I did this was to see how much I could improve the performance over some other OpenCVSharp wrappers using a ThreadPool class and provide a good API.

It runs at ~20FPS on my i7-4770/GTX 770M and at ~80FPS on my new i5-7500/GTX 1080 so it is almost realtime for high end desktops. Not sure how much CV can be done using shaders but this is a decent way for me to "ease into" OpenCV as I have always found setup a daunting task.
